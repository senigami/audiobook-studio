# Security Policy

## Supported Versions

Audiobook Studio is currently maintained on the latest release line only.

| Version | Supported |
| ------- | --------- |
| 1.6.x   | Yes       |
| < 1.6.0 | No        |

Older versions may contain known issues that have already been addressed in the current release. Security fixes are not currently backported to older release lines.

## Reporting a Vulnerability

If you believe you have found a security vulnerability, please do not open a public issue.

Instead, use GitHub's private vulnerability reporting flow for this repository:

- Open the repository's **Security** tab
- Click **Report a vulnerability**
- Include as much detail as you can, such as:
  - a clear description of the issue
  - steps to reproduce
  - affected versions or branches
  - proof of concept, logs, screenshots, or sample requests if relevant
  - any suggested remediation if you have one

What to expect:

- I will try to acknowledge valid reports within a few days
- I may ask follow-up questions to confirm scope, impact, or reproduction steps
- If the report is accepted, I will work on a fix and coordinate disclosure as responsibly as possible
- If the report is declined, I will explain why when I can

Please avoid publishing full exploit details publicly until the issue has been reviewed and a fix or mitigation is available.
